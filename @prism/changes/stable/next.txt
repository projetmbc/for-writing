[[:api:]]
Here are the breaking \chges.

    1) \lua \prod: \func lua::''getPal'' is now lua::''palCreateFromPal''. It expects a palette \var instead of a name alongside options.

    1) \Contribs workflow has been fully changed (see below).

    1) No more same size for all the palettes (see below).

    1) The categories have been fully changed (see below).


[[:doc:]]
Manual: remove the cluster appendix.


-----
:new:
-----

[[:api:]]
Here are the new features.

    1) 43 new palettes added.

    1) Multiple \distrib formats are now available to meet various technical requirements.

        + High-fidelity (unmodified).

        + Normalized (''size <= MAX_SEMANTIC_SIZE'').

        + Modular (one file per palette).

        + Monolithic (all palettes in one file).

    1) \Categos: the size palette is no longer used since the new \api provides named \catgeos. For example, a 3-color smooth palette can be now sequential, qualtitaive and/or semantic.

    1) \Contribs workflow simplified significantly.

        + Data extraction and code generation are now streamlined by instantiating the python::''PaletteTransformer'' class, alongside palette and \api builder functions.

        + Monolithic or modular version can be omitted (\tex can't build all the high-fidelity palettes).

        + Standardized showcases now feature simplified graphics: waves for small palettes and random triangles for large ones.
