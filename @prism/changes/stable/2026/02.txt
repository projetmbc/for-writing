-------
:break:
-------

[[:api:]]
Here are the breaking \chges.

    1) Palette names use "semantic" camelcase. For example, we use ''PinkGrey'' and ''BWR'' (Blue White Red) instead of ''Pinkgrey'' and ''Bwr'', but we keep ''RdBu'' (RedBlue).

    1) \lua \prod: \func lua::''getPal'' is now lua::''palCreateFromPal''. It expects a palette \var instead of a name alongside options.

    1) \Contrib workflow has been fully changed (see below).

    1) No more same size for all the palettes (see below).

    1) No sublist or shifted palette.

    1) The categories have been fully changed (see below).


[[:doc:]]
Manual.

    1) Ignored and renamed palettes have been placed in dedicated appendices.

    1) Remove the cluster appendix.


-----
:new:
-----

[[:api:]]
Here are the new features.

    1) 95 new palettes added.

    1) Multiple \distrib formats are now available to meet various technical requirements.

        + High-fidelity (unmodified).

        + Normalized (''size <= MAX_SEM_SIZE'').

        + One file per palette (no more all palettes in a single file).

    1) \Categos: the size palette is no longer used since the new \api provides named \categos. For example, a 3-color smooth palette can be now sequential, qualtitaive and/or semantic.

    1) \Contrib workflow simplified significantly.

        + Data extraction and code generation are now streamlined by instantiating the python::''PaletteTransformer'' class, alongside palette and \api builder functions.

        + Standardized showcases now feature simplified graphics.
